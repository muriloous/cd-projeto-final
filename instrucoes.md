Instruções
O objetivo deste projeto é que você aplique os principais conceitos de Ciência de Dados básicos para criar uma solução completa, desde a obtenção dos dados até a apresentação de insights e análises relevantes para a algum público. Esse público pode ser a população em geral, com apresentação de dados sobre saúde, políticas públicas, etc., ou até mesmo, para um público com interesses banais como aficionados por cinema ou futebol. Você deverá desenvolver um dashboard Interativo utilizando a biblioteca Streamlit como base de criação da interface, que apresente dados de forma clara e visualmente atraente.

Tema:
Escolha um tema do interesse do grupo (tema totalmente livre!) que utilize dados para levantar e responder a questões relevantes para a sociedade. O projeto deve demonstrar como a análise de dados pode gerar análises e informar o público ou decisores.

Requisitos funcionais do projeto:
Seu projeto deve obrigatoriamente incluir os seguintes elementos e tecnologias:
1 - Conceitos de Ciência de Dados:
Coleta e armazenamento: Uso de API de Dados para obtenção de dados públicos (Ex: dados governamentais, APIs de serviços, etc.).
Armazenamento em arquivo: Os dados obtidos (ou uma versão processada deles) devem ser armazenados localmente em um arquivo (.csv, .json, etc.) antes de serem carregados pelo dashboard.
Processamento/Limpeza de dados: Aplicação de técnicas básicas de limpeza e transformação (filtragem, agregação, criação de novas colunas, etc.).
Análise exploratória e visualização: Geração de gráficos e métricas que ajudem a entender e comunicar os insights do seu tema.
2 - Interface e dashboard:
Streamlit: A interface do usuário deve ser construída exclusivamente com a biblioteca Streamlit.
Interatividade: O dashboard deve possuir pelo menos dois elementos interativos (Ex: slider, caixa de seleção, dropdown, botão de rádio, etc.) que permitam ao usuário explorar e filtrar os dados dinamicamente.
Layout: O dashboard deve ter um layout organizado, com títulos claros, explicações sobre os gráficos e métricas de destaque (KPIs).

Entrega:
O grupo poderá ser formado por até 6 integrantes
A entrega final será feita via GitHub e deve consistir em:
Código-Fonte: Todos os arquivos de código (.py), o arquivo de requisitos (requirements.txt) e a pasta com os dados brutos/processados.
Documentação (README): Um arquivo README.md no repositório que contenha:
O título e o tema do projeto, com uma breve justificativa do escolha do tema pela equipe.
A fonte da API de Dados utilizada e uma breve descrição dos dados.
As perguntas-chave que o dashboard se propõe a responder.
Instruções de como rodar o projeto localmente (ex: streamlit run app.py).
Capturas de tela do dashboard finalizado, com explicação de seu contexto.
